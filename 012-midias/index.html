Mídias e áudios

Imagem dinâmica

Tente abrir esse site em vários dispositivos diferentes ou simplesmente aumente e diminua o tamanho do seu navegador.

        
            <picture>
                <source media="(max-width: 750px)" srcset="imagens/foto-p.png" type="image/png">
                <source media="(max-width: 1000px )" srcset="imagens/foto-m.png" type="image/png">
                <img src="imagens/foto-g.png" alt="Imagem flexível">
            </picture>

Assim, colocando as imagens da menor pra para a maior, dependendo do tamanho do dispositivo ou navegador, ela irá carregar a imagem de acordo.

Para experimentar, movimente o tamanho deste navegador para ver as diferentes imagens de tamanhos diferentes.

Lembrando que a ordem da escrita do HTML tem que obedecer da menor imagem (vem primeiro) para a maior imagem.

Áudio

Vamos aprender a reproduzir áudios em HTML5

            <audio src="audio/audio1.mp3" controls autoplay></audio>

Os dados do áudio devem ser preenchidos assim como nas imagens, mas com os parâmetros "controls" e "autoplay" dentro da tag <audio>

Áudios que podem ser carregados na maioria dos navegadores: .mp3, .wav, .ogg dentre outros

Temos uma outra maneira:



            
                <audio preload="metadata" autoplay controls loop>
                    <source src="audio/audio1.mp3" type="audio/mpeg">
                    <source src="audio/audio1.ogg" type="audio/ogg">
                    <source src="audio/audio1.wav" type="audio/wav">
                </audio>


=metadata para que ele não reproduza todo o audio antes do comando do usuário.

=autoplay para que o audio seja carregado mas não seja reproduzido de imediato, aguardando o comando do usuário.

=controls para que o player fique disponível na tela para o usuário manipular.

=loop para que o áudio repita até que o usuário pare o áudio no player.

As outras opções - se voce tiver - devem ser carregadas na mesma pasta de arquivos, isso resolve o problema de diferentes tipos de navegadores. Caso um não seja reproduzido, o outro será.

Inserindo Vídeos localmente

ARQUIVOS MP4, M4V (PEQUENO MAS DE BOA QUALIDADE) E OGV HOSPEDADOS NO MEU PRÓPRIO SERVIDOR.

 

            
                <video poster="imagens/img.png" src="video/videomp4.mp4 " controls autoplay loop width="200">
                    <source src="video/videowebm.webm" type="video/webm">
                    <source src="video/Videom4v.m4v" type="video/m4v">
                    <source src="video/videomp4.mp4" type="video/mp4">
                </video>

Observação: Essa prática pode ser muito caro para seu cliente pois pode consumir um alto número de banda dependendo do acesso que o site dele tiver. ATENÇÃO!!!

### = < Caso o navegador não carregue nenhum dos arquivos, uma mensagem você deve escrever em uma tag <p> com este aviso, pois o vanegador vai tentar reproduzir todos os arquivos da lista, se ele não conseguir, a imagem é exibida.

### = < Note que o atributo "poster" foi colocado dentro da tag <video>, isso para que no lugar do vídeo seja mostrado uma tumbnail (melhor imagem que descreve o vídeo) pra chamar a atenção do usuário lhe dando uma prévia do conteúdo.

Hospedagem Externa YOUTUBE

Veremos uma forma de hospedagem com maior custo benefício:

Uma forma bem melhor, mais fácil e com menos consumo de banda é colocar o link do vídeo já hospedado externamente, nesse caso usaremos o Youtube.
Simples!

Pra isso basta pegar o link dentro da função de "compartilhar" do vídeo do youtube, na função "incorporar" copie o código gerado e cole direto no código HTML.

        
            <iframe width="560" height="315" src="https://www.youtube.com/embed/fTQdUtvgbhQ" title="YouTube
            video player" frameborder="0" allow="accelerometer; 
            autoplay; clipboard-write; encrypted-media;  gyroscope; picture-in-picture" allowfullscreen></iframe>

Desta forma você não consome dados do servidor próprio e sim do próprio Youtube.

Insentive seu cliente a ter conteúdos de vídeos no Youtube para melhorar a navegação e hospedagem.

Caso seu cliente queira que esse vídeo só esteja disponível para visualização no site, escolha fazer esse procedimento com o Vimeo.

Lá é oferecido a opção de deixar os vídeos privados para que os sites de busca não o carreguem se ele estiver privado.

O procedimento de carregar o link é o mesmo, porém a hospedagem do Vímeo é paga
